# 🧠 Project Structure — Graph-Based Social Connection System

# =========================================================

# 🧱 TECH STACK (Suggested)

# =========================================================

Backend     : Node.js (TypeScript) OR Python (FastAPI)
Frontend    : React (Next.js + TypeScript)
Database    : Neo4j (Graph DB) OR PostgreSQL + Redis
Realtime    : WebSockets / Socket.io
Auth        : JWT / OAuth (Google, etc.)
DevOps      : Docker + Nginx

---

# 📂 ROOT STRUCTURE

# =========================================================

project-root/
│
├── backend/              # Core logic, APIs, graph processing

# =========================================================

# ⚙️ BACKEND STRUCTURE

# =========================================================

backend/
│
├── src/
│   ├── controllers/      # Handle API requests (user, graph, path)
│   ├── services/         # Business logic (pathfinding, scoring)
│   ├── models/           # Data models (User, Edge)
│   ├── routes/           # API route definitions
│   ├── middleware/       # Auth, validation, logging
│   ├── graph/            # Graph algorithms (BFS, Dijkstra)
│   ├── utils/            # Helper functions
│   ├── config/           # DB, env configs
│   └── app.ts            # App entry point
│
├── tests/                # Unit + integration tests
├── package.json          # Dependencies
└── tsconfig.json         # TypeScript config

---

# =========================================================

# 🔥 BACKEND KEY MODULES

# =========================================================

graph/
│
├── bfs.ts                # Shortest path (exploration)
├── dijkstra.ts           # Weighted path (best intro)
├── constraints.ts        # REAL→DEMO→REAL restriction logic
└── scoring.ts            # Confidence + trust scoring

services/
│
├── path.service.ts       # Core path query logic
├── user.service.ts       # User creation + linking
├── graph.service.ts      # Graph operations (add/remove edges)
└── demo.service.ts       # Demo graph handling

# =========================================================

# 🧩 DATA MODEL (SIMPLIFIED)

# =========================================================

User:

* id
* name
* type (REAL / DEMO)
* metadata (branch, year, etc.)

Edge:

* from
* to
* isReal (true/false)
* weight (optional)
