# 🧠 Graph-Based Social Connection System (Demo + Real Hybrid)

## 📌 Core Idea

The system is based on the concept of **small-world networks** — the idea that most people are connected through a short chain of relationships.

Goal:

* Help users discover **how they are connected to someone**
* Enable **referrals, introductions, and networking**
* Provide **intelligent path suggestions**, not just raw connections

Unlike platforms like LinkedIn or Instagram, this system explicitly shows connection paths and optimizes them.

---

## ⚙️ Graph Model

* Users = Nodes
* Connections = Edges

Two types of nodes:

* REAL_NODE → actual users
* DEMO_NODE → synthetic users (for bootstrapping)

Two types of edges:

* REAL_EDGE → real connections
* DEMO_EDGE → simulated connections

Optional:

* Weight on edges (trust, interaction strength)

---

## 🚀 Hybrid System (Demo + Real Together)

To solve cold start problem:

* Start with ~20,000 demo users
* Each with ~50 connections (≈ 500,000 edges)
* Graph follows **small-world properties**:

  * Clusters (departments, clubs)
  * Short path lengths (2–3 hops)

As real users join:

* They are added as REAL_NODE
* They form REAL_EDGE connections

---

## ⚠️ Critical Design Rule (MOST IMPORTANT)

❌ Never allow:
REAL → DEMO → REAL paths

Reason:

* Creates fake reachability
* Breaks trust
* Misleads users

---

## ✅ Correct Graph Constraints

Allowed:

* REAL → REAL → REAL
* DEMO → DEMO → DEMO
* REAL → DEMO (exploration only)

Not allowed:

* REAL → DEMO → REAL ❌

---

## 🧠 Dual Mode System

### 🔍 Exploration Mode

* Uses: REAL + DEMO graph
* Purpose:

  * Visualize network
  * Discover possible reach
* Demo edges shown as grey

---

### ⚡ Action Mode

* Uses: REAL graph only
* Purpose:

  * Referrals
  * Messaging
  * Introductions

---

## 🧩 Pathfinding Logic

Algorithms used:

* **Breadth-first search** → shortest path
* **Dijkstra's algorithm** → weighted best path

---

### Path Constraint Rule

During traversal:

```
If node == DEMO:
    cannot move back to REAL
```

This ensures:

* No fake bridging
* Demo stays visual only

---


## 🏫 College-Level Implementation (50k scale)

### Dataset

* Nodes: 20,000–50,000
* Avg degree: ~50
* Edges: ~500k–2M

---

### Step 1: Generate Demo Graph

* Divide into communities:

  * Branches (CSE, ECE, etc.)
  * Years
  * Clubs

Connections:

* ~30 local (same group)
* ~20 random (cross-group)

Result:

* Realistic **small-world network**

---

### Step 2: Add Real Users

* User registers → becomes REAL_NODE
* Connections added via:

  * Contacts
  * Mutual confirmations
  * Manual adds

---

### Step 3: Edge Rules

Database logic:

```sql
DELETE FROM edges
WHERE type = 'DEMO'
AND node1.isReal = TRUE
AND node2.isReal = TRUE;
```

---

### Step 4: Graph Storage

Options:

* Adjacency list (C++ / backend)
* Graph DB (Neo4j style)

Edge structure:

```cpp
struct Edge {
    int to;
    bool isReal;
    float weight;
};
```

---

### Step 5: Path Query

#### Exploration Mode:

* Use full graph
* BFS

#### Action Mode:

* Use real-only graph
* BFS / Dijkstra

---
